User Need

I need to be able to find features and content easily.

Related User Story: Searchable.

What to Do

Provide a friendly search capability. Ideally search should include:

  • Assistance to start a search, such as:
    • autocomplete,
    • ability to easily find previous searches, and
    • spell-check,
    • allowing variations of search terms. For example: Caps or no caps, singular or plural forms, acronyms, or similar terms.
  • Support for finding useful results (findability), such as:
    • grouping results when appropriate, with headings for each group,
    • useful labels and descriptions for each result,
    • filters to refine users’ search queries, when appropriate,
    • Boolean search operators and modifiers, and explain how to use them, and
    • ability to customize the number and presentation of links in search results.
  • Reliable search
    • Test the search function with real users. Make sure the search works reliably, and finds results that match user intent for common searches.
How it Helps

Having a search capability allows users to find the content they need, even if they cannot use the site menus. Many users can not manage menu systems for a variety of reasons. For example, menu systems and most site navigation require the user to understand the menu categories. Some users may be unable to identify the correct categories. However, many users learn how to use search and reuse that skill on many sites.

Search is most useful when it makes it easier for users to find what they are looking for. At the input level, this includes correcting misspellings or suggesting auto-corrected versions of the search terms. Accepting variations of search terms benefits many people, including people with language impairments, dyslexia, or a memory impairment.

Allowing users to narrow their search prevents them from getting inundated with irrelevant results. This is especially important for slow readers with impaired memory or users who get easily overwhelmed.

The next step is helping users easily find useful results. Users with impaired short-term memory, age related forgetfulness, or who are easily distracted may find it difficult to navigate a site and look through many results. If it takes too long, they may lose focus and forget what they are looking for. A busy page of search results containing long lists of links can also be overwhelming. For example, people with obsessive-compulsive disorder (OCD) sometimes feel a need to open and check every result, even when most of them are not useful.

It helps if search results are presented under headings and categories. Advanced search options and search filters also offer users greater control over search queries, making it easier to find the right information. Providing an easy customization function that allows users to set the number of links and modify their presentation in search results makes it easier for users to recognize the correct search result and avoid becoming overwhelmed.

Finally, make sure that search options function reliably, consistently, and as expected.

Errors and frustration can worsen anxiety, which in turn reduces overall cognitive function. This means that by the time the user finds the page or task they are looking for, they are less likely to be able to complete it. Making the process of finding the right page easier will help them complete the task.

More Details

Search is less important on small sites where every page is no more than two clicks away from the main page.

Examples

Use:

  1. Search that forgives or corrects spelling mistakes and allows similar terms such as acronyms or related terms. For example:
    • The user asks for information about “adhd” but uses lowercase. Information about ADHD, ADD, AD(H)D, and attention-related disabilities were also included in the results.
  2. Search that helps the user narrow the results to the most useful. For example:
    • A supermarket supports searching for chocolate by providing filters to reduce the search results such as vegan, gluten-free, sugar-free, etc.
    • The supermarket allows users to add Boolean terms, such as “chocolate and syrup.” The site makes it easy to find instructions on how to use these terms to narrow your search.
    • Search results are presented under specific headings that make it easier to find the relevant results.
  3. Personalisation so that the users can customize the number and presentation of links in search results.
  4. Search that has successful user testing with people with learning and cognitive disabilities find it easy to find the correct content.

Avoid:

  1. Search that overwhelms the user with results that are not what they are looking for. For example:
    • A search has lots of results but the user has no way to filter them.
  2. Search that is not flexible and does not correct mistakes. For example:
    • The search does not support similar terms or incorrect spelling
  3. Search that does not have useful labels or descriptions for each search result. The user has to open the results to know if they are relevant.
